# 📦 microservice_conect

## 🧭 Visão Geral

Este projeto faz parte de um estudo prático de **arquitetura de microservices**, com foco na comunicação entre serviços independentes.

O objetivo é simular um ambiente distribuído onde diferentes serviços se conectam para troca de dados e execução de operações de forma desacoplada.

---

## 🔗 Relação com outros projetos

Este repositório faz parte de um ecossistema de estudos sobre microservices.

Ele evolui conceitos explorados no projeto `microservices-study1`, servindo como uma etapa mais prática e voltada para integração entre serviços.

- `microservices-study1` → base de aprendizado e experimentação
- `microservice_conect` → aplicação mais focada em comunicação entre serviços

---

## 🧠 Conceitos Aplicados

- Arquitetura de microservices
- Comunicação entre serviços via HTTP/REST
- Independência entre aplicações
- Integração de APIs
- Separação de responsabilidades

---

## 🛠️ Tecnologias Utilizadas

- Java
- Spring Boot
- Spring Web (REST APIs)
- Maven
- Git

---

## ⚙️ Funcionalidade

O projeto simula a comunicação entre serviços independentes, onde cada serviço possui uma responsabilidade específica e pode interagir com outros serviços via chamadas HTTP.
